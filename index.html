Here we will introduce some basics of R, followed by a brief overview of linear regression. The materials are taken from MITx 15.071x - The Analytics Edge and Introduction to Statistical Learning.

Basics

All data in R are represented in vectors. For example:

d <- c(10,20,30,40)

Here, we used the c function, standing for “combine”. It returns a new vector of size 4.

One great thing about R is the documentation. We can access it quickly with ?:

?c

In R, indexing starts with 1, unlike a lot of other languages, which start with 0:

d[1]
## [1] 10

Functions

Let’s have a look at the few of the common functions.

First, we have str, which displays the structure of an object:

str(d)
##  num [1:4] 10 20 30 40

This tells us that we have a numeric vector of 4 elements.

To get the summary statistics, use summary:

summary(d)
##    Min. 1st Qu.  Median    Mean 3rd Qu.    Max. 
##    10.0    17.5    25.0    25.0    32.5    40.0

We can also get the same numbers with mean, max, min:

mean(d)

max(d)
## [1] 40
min(d)
## [1] 10

Data

This dataset is taken from MITx 15.071x - The Analytics Edge.

The dataset contains climate data from May 1983 to December 2008. The available variables include:

  • Year: the observation year.

  • Month: the observation month.

  • Temp: the difference in degrees Celsius between the average global temperature in that period and a reference value. This data comes from the Climatic Research Unit at the University of East Anglia.

  • CO2, N2O, CH4, CFC.11, CFC.12: atmospheric concentrations of carbon dioxide (CO2), nitrous oxide (N2O), methane (CH4), trichlorofluoromethane (CCl3F; commonly referred to as CFC-11) and dichlorodifluoromethane (CCl2F2; commonly referred to as CFC-12), respectively. This data comes from the ESRL/NOAA Global Monitoring Division.

  • CO2, N2O and CH4 are expressed in ppmv (parts per million by volume – i.e., 397 ppmv of CO2 means that CO2 constitutes 397 millionths of the total volume of the atmosphere)

  • CFC.11 and CFC.12 are expressed in ppbv (parts per billion by volume).

  • Aerosols: the mean stratospheric aerosol optical depth at 550 nm. This variable is linked to volcanoes, as volcanic eruptions result in new particles being added to the atmosphere, which affect how much of the sun’s energy is reflected back into space. This data is from the Godard Institute for Space Studies at NASA.

  • TSI: the total solar irradiance (TSI) in W/m2 (the rate at which the sun’s energy is deposited per unit area). Due to sunspots and other solar phenomena, the amount of energy that is given off by the sun varies substantially with time. This data is from the SOLARIS-HEPPA project website.

  • MEI: multivariate El Nino Southern Oscillation index (MEI), a measure of the strength of the El Nino/La Nina-Southern Oscillation (a weather effect in the Pacific Ocean that affects global temperatures). This data comes from the ESRL/NOAA Physical Sciences Division.

Subsetting

Let’s start by taking a single observations:

climate[1,1]
## [1] 1983

This gives us the first observation of the first column.

We can take logical selections of data, like so:

climate[climate$Year > 2000,]

Notice that since we want to get all the columns, we can leave the column selection empty, which defaults to all. A common mistake would be to do this:

climate[climate$Year > 2000]

Training and testing data

Let’s separate our dataset in two parts. We will use points from 1983 through 2005 as training data, and 2006 through 2008 as test data.

Thus, we will train our model on the training of the data, and then attempt to predict the test data.

climateTrain <- climate[climate$Year <= 2005,]
climateTest <- climate[climate$Year >= 2006,]

Linear regression

Linear regression is probably the most simple supervised learning algorithm. It finds a line to fit through the data. This is done by minimizing the total distance between each of the points and the proposed line.

R integrates linear regression on a very basic level. Let’s start with an example.

We want to see if the annual aerosol level is directly proportional with the temperature difference.

model1 <- lm(Temp ~ Aerosols, data = climateTrain)

Here, we are predicting Temp based on Aerosols, or regressing Temp on Aerosols. For now, let’s plot the two sets of observations, and use the line of best fit that R calculated:

plot(climateTrain$Aerosols, climateTrain$Temp, pch=20)
abline(model1, col='red')

We can see that the data is rather non-linear, but it is possible draw some conclusions with a straigh-line fit. How did our model do?

Model summary

First, the residuals. This the summary of the distances of all the points to the line of best fit. This is just like summary of a vector.

Coefficients are the linear equation coefficients. Since we are doing single variable regression, ax+b, we get values for Intercept, which is b, and (in our case) Aerosols, which is a.

Significance

The first column is Estimate, which is our best (least-squares) guess for these values.

We then have the standard error (Std. Error), which describes what the spread of this value is.

t value is the ratio of the two. Since we usually assume that error is normally distributed (since we don’t know any better), we use can use a \(t\)-test to figure our how confident we are in the prediction.

We want to know if our estimate is meaningful. The worst case scenario is that the two values we are comparing (Temp and Aerosols) have nothing to do with each other. If this is the case, the slope of the best fitting line would be 0. We thus use the \(t\)-test to compare the value of the slope we found to zero.

The probability column tells us what the chance of observing the slope value by chance alone (according to our normality assumptions, that is). The lower this probability is, the more confident we are that the slope is different from zero.

Regression coefficient

How well does our model prodict the data? We can calculate \(R^2\), which is a measure of goodness of fit.

\[R^2 = 1 - \frac{{SS}_{res}}{{SS}_{tot}} \]

Here, \({SS}_{res}\) is the sum of residuals from our best line.

\[{SS}_{res} = \sum(y_i-f_i)^2\]

\({SS}_{tot}\) is the sum of residuals from a line with a slope 0.

\[{SS}_{tot} = \sum(y_i-\bar{y})^2\]

Thus, \(R^2\) is a measure of fit of our line against a base-case line. \(R^2\) of 0 means that the base line and our line are essentially the same. \(R^2\) of 1 means very high linear correlation between the data and our line.

The great thing about \(R^2\) is that for the training dataset, we will always have it between 0 and 1. A low \(R^2\) is not necessarily bad. If know that some data can not be linear, and having high \(R^2\) is a sign of toruble. For some data, which is a sum of multiple components, we can not reach high \(R^2\) due to noise.

Note that for our data, we have a rather low \(R^2\). This is echoed by the fact that the data is hard to describe by a single line.

Prediction

Now that we have derived a model, we can try to predict the values in the test dataset.

prediction <- predict(modelAIC, climateTest)

Let us write a function that calculates \(R^2\) for the test dataset:

predictedCoefficientOfDetermination <- function(model, testData, respondingVariable) {
  prediction <- predict(model, testData)
  ssRes <- sum((respondingVariable - prediction)^2)
  ssTot <- sum((respondingVariable - mean(respondingVariable))^2)
  return (1 - (ssRes/ssTot))
}

Now we can use the newly minted function to determine how well our model does on the test data:

predictedCoefficientOfDetermination(modelAIC, climateTest, climateTest$Temp)
## [1] 0.282211
